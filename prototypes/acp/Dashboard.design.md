# Dashboard 设计说明

## 布局

```
┌─────────────────────────────────────────────┐
│  Header (56px): Logo + Nav + User           │
├─────────────────────────────────────────────┤
│  统计栏: [在线Agent] [今日任务] [活跃工作流]   │
├─────────────────────────────────────────────┤
│  Agent 卡片网格                               │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐       │
│  │ Lyra │ │  PM  │ │  UX  │ │Alice │       │
│  └──────┘ └──────┘ └──────┘ └──────┘       │
└─────────────────────────────────────────────┘
```

## 关键决策

### 卡片网格 vs 表格
选择**卡片网格**。理由：
- Agent 数量少（4-8个），表格浪费空间
- 卡片能承载更丰富的状态信息（图标、进度条、颜色）
- 监控面板视觉上更直观

### 响应式断点
| 断点 | 每行卡片数 |
|------|-----------|
| xl ≥1200 | 4 |
| lg ≥992 | 3 |
| sm ≥576 | 2 |
| xs <576 | 1 |

### 统计栏
使用 AntD `Statistic` + 左侧色条（borderLeft 3px），三列等宽。数值大号加粗，标题带图标。

### 卡片内容层级
1. **名称 + 角色标签 + 状态灯**（最醒目）
2. **模型信息**（次级灰色文字）
3. **当前操作**（浅色背景区块，突出动态信息）
4. **Token + 最后活跃**（底部辅助信息）

### 排序规则
working > online > error > offline。离线 agent 透明度降至 55%。

## 状态处理

| 状态 | 表现 |
|------|------|
| 加载中 | 4个 Skeleton 卡片占位 |
| 空数据 | Empty 组件 "暂无 Agent" |
| 加载失败 | 顶部 Alert 红色错误条 |
| 正常 | 卡片网格 + 10s 静默刷新 |

## 交互

- 卡片 hover 有阴影抬起效果（AntD hoverable）
- 点击卡片 → 跳转 `/agents/:id`
- 刷新无闪烁（静默替换数据，不重新 loading）
- Token 进度条 hover 显示精确数字
